# Dropbox 能力差距与实施优先级

## 目的

本清单以用户管理云端同步文件的实际流程为中心，对比成熟云盘常见能力，不以功能数量作为目标。结论基于当前 SyncHub 的 API、CLI、数据模型和 React 管理端实现；Dropbox 的具体套餐限制和保留时长不作为本项目的兼容目标。

## 当前能力

| 领域 | 当前状态 | 说明 |
| --- | --- | --- |
| 多端文件同步 | 已具备 | CLI workspace、变更游标、设备心跳和同步 daemon。 |
| 网页文件管理 | 已具备 | React 管理端支持浏览、上传、下载、新建目录、移动/重命名和删除。 |
| 冲突处理 | 已具备 | 服务端记录版本冲突，网页端可选择保留本地、云端或两者。 |
| 文件版本 | 已具备 | API、CLI 和网页端支持查看、固定及恢复文件版本。 |
| 本地删除保护 | 已具备 | CLI pull 会将远端删除的本地文件移动到工作区 `.synchub/trash`。 |
| 运行状态 | 已具备 | `/readyz` 检查 PostgreSQL 和 storage，管理端显示设备最近在线时间。 |
| 活动记录 | 已具备 | 管理端可查看当前用户文件的创建、更新、移动、删除与恢复时间线。 |
| 安全预览 | 已具备 | 管理端限制大小后预览文本、常见图片和 PDF，不内嵌 HTML、SVG 或脚本。 |
| 多文件上传 | 已具备 | 管理端支持多选文件、4 MiB 分块、逐项上传进度、失败隔离和单项重试。 |

## 高优先级缺口

### 1. 云端回收站

文件节点已使用 `deleted_at` 软删除，但没有列出已删除节点或恢复节点的服务端 API。网页删除后用户只能依赖已有版本数据或本地 workspace trash，无法在网页端找回整个文件或目录。

需要交付：

- 已删除文件/目录列表，按删除时间分页。
- 恢复单个文件或目录，处理同路径冲突。
- 删除目录时的子树恢复语义。
- 保留期、彻底删除和存储回收策略。

### 2. 分享与权限

当前所有文件都只属于一个用户，缺少团队协作的核心边界。需要在回收站完成后再设计，避免权限模型与删除恢复相互冲突。

需要交付：

- 文件夹成员、只读/编辑角色与移除成员。
- 可过期的公开分享链接及下载限制。
- 共享根目录与个人目录之间的权限继承规则。
- 分享、下载和权限变更的审计事件。

### 3. 检索与预览

名称/路径搜索和受限文件预览已经完成，后续可在保持安全边界的前提下扩展更多格式。

需要交付：

- 服务端分页搜索 API 已覆盖文件名和路径。
- 管理端支持文本、常见图片和 PDF 的大小受限预览。
- HTML、SVG、脚本和未知格式不会在浏览器中内嵌执行。

## 中优先级缺口

| 能力 | 当前限制 | 建议方向 |
| --- | --- | --- |
| 存储配额 | 无用户级容量上限或用量统计 | 汇总逻辑大小和对象大小，上传前执行配额检查。 |
| 文件上传体验 | 已支持多文件队列、分块上传、进度和失败重试，暂不支持目录上传与传输中取消 | 后续增加目录选择、分块并发及取消上传会话。 |
| 设备控制 | 仅显示设备 | 支持撤销设备、显示同步错误和最后一次成功同步。 |
| 通知 | 无 | 对冲突、共享邀请、配额临近和同步失败发送站内通知。 |
| 保留策略 | 版本清理由全局配置控制 | 将版本与回收站策略明确为用户或组织级策略。 |

## 非当前优先级

- 在线协作文档编辑。
- 第三方应用集成和自动化工作流。
- 大规模团队治理、DLP、eDiscovery。
- 跨区域高可用；当前 local storage + 单 Fly Machine 不支持安全横向扩展。

## 推荐顺序

1. 云端回收站和恢复 API。
2. 文件名/路径搜索。
3. 存储用量和配额。
4. 分享与权限模型。
5. 预览、活动记录和通知。

每一项在开始前都要确认 PostgreSQL 与 SQLite 实现、同步 change event、OpenAPI、CLI 和 React 管理端是否需要同时更新。
