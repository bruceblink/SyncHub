# Dropbox 能力差距与实施优先级

## 目的

本清单以用户管理云端同步文件的实际流程为中心，对比成熟云盘常见能力，不以功能数量作为目标。结论基于当前 SyncHub 的 API、CLI、数据模型和 React 管理端实现；Dropbox 的具体套餐限制和保留时长不作为本项目的兼容目标。

## 当前能力

| 领域 | 当前状态 | 说明 |
| --- | --- | --- |
| 多端文件同步 | 已具备 | CLI workspace、变更游标、设备心跳和同步 daemon。 |
| 网页文件管理 | 已具备 | React 管理端支持稳定游标分页浏览和搜索，并可上传、下载、新建目录、移动/重命名和删除。 |
| 冲突处理 | 已具备 | 服务端记录版本冲突，网页端可选择保留本地、云端或两者。 |
| 文件版本 | 已具备 | API、CLI 和网页端支持查看、固定及恢复文件版本。 |
| 本地删除保护 | 已具备 | CLI pull 会将远端删除的本地文件移动到工作区 `.synchub/trash`。 |
| 运行状态 | 已具备 | `/readyz` 检查 PostgreSQL 和 storage，管理端显示设备最近在线时间。 |
| 活动记录 | 已具备 | 管理端可查看当前用户文件的创建、更新、移动、删除与恢复时间线。 |
| 安全预览 | 已具备 | 管理端限制大小后预览文本、常见图片和 PDF，不内嵌 HTML、SVG 或脚本。 |
| 多文件上传 | 已具备 | 管理端支持多选文件与文件夹、保留目录结构、4 MiB 分块、逐项进度、主动取消、失败隔离和单项重试。 |
| 云端回收站 | 已具备 | 管理端可通过稳定游标分页查看、恢复或彻底删除文件与目录树，显示自动保留期，并处理恢复路径冲突。 |
| 存储用量与配额 | 已具备 | 服务端统计用户逻辑文件用量，在上传会话创建前校验可配置配额；管理端显示用量、上限和剩余空间。 |
| 设备控制 | 已具备 | 管理端显示同步设备及最近在线状态，并支持撤销其他设备。 |

## 高优先级缺口

### 1. 分享与权限

当前所有文件都只属于一个用户，缺少团队协作的核心边界。需要在回收站完成后再设计，避免权限模型与删除恢复相互冲突。

需要交付：

- 文件夹成员、只读/编辑角色与移除成员。
- 可过期的公开分享链接及下载限制。
- 共享根目录与个人目录之间的权限继承规则。
- 分享、下载和权限变更的审计事件。

### 2. 回收站保留与对象回收策略

回收站列表、恢复、用户主动彻底删除和全局自动保留期已经完成。彻底删除和历史版本清理会把不可变对象写入持久化 GC 队列；后台任务延迟复查引用、与相同哈希上传互斥，并对失败任务指数退避重试。商业化阶段还需支持套餐级保留策略。

### 3. 检索与预览

名称/路径搜索和受限文件预览已经完成，后续可在保持安全边界的前提下扩展更多格式。

需要交付：

- 服务端分页搜索 API 已覆盖文件名和路径。
- 管理端支持文本、常见图片和 PDF 的大小受限预览。
- HTML、SVG、脚本和未知格式不会在浏览器中内嵌执行。

## 中优先级缺口

| 能力 | 当前限制 | 建议方向 |
| --- | --- | --- |
| 存储策略 | 配额目前是全局默认值 | 商业化时将配额和保留策略迁移为用户或组织套餐配置。 |
| 文件上传体验 | 已支持文件夹、多文件队列、分块上传、进度、取消和失败重试；浏览器不会提供纯空目录 | 后续根据服务压力评估受控分块并发，并视需求补空目录导入。 |
| 设备状态 | 已支持显示、撤销设备，以及最近同步成功/失败时间与错误摘要 | 后续可按设备保留短期同步运行历史。 |
| 通知 | 无 | 对冲突、共享邀请、配额临近和同步失败发送站内通知。 |
| 保留策略 | 版本清理由全局配置控制 | 将版本与回收站策略明确为用户或组织级策略。 |

## 非当前优先级

- 在线协作文档编辑。
- 第三方应用集成和自动化工作流。
- 大规模团队治理、DLP、eDiscovery。
- 跨区域高可用；当前 local storage + 单 Fly Machine 不支持安全横向扩展。

## 推荐顺序

1. 分享与权限模型。
2. 用户或组织套餐级配额。
3. 通知。

每一项在开始前都要确认 PostgreSQL migration/repository、同步 change event、OpenAPI、CLI 和 React 管理端是否需要同时更新。
