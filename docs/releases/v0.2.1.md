# SyncHub v0.2.1 Release Notes

Date: 2026-07-12

SyncHub v0.2.1 is the published server release for the PostgreSQL-only v0.2 series. It retains the server, React administration console, upload, synchronization, retention, and storage capabilities prepared for v0.2.0 while repairing the release pipeline that prevented those artifacts from being published.

## Fixes

- Made the Docker image smoke test start an isolated PostgreSQL 17 container when no external `DATABASE_URL` is supplied.
- Verified embedded migrations, database and storage readiness, and runtime version reporting without production database credentials.
- Pinned the web administration build to pnpm 10.30.2 and enabled deterministic non-interactive Docker builds.
- Excluded host `node_modules` and generated frontend output from the Docker context so Windows dependencies cannot overwrite Linux container dependencies.

## Upgrade Notes

1. Back up the PostgreSQL database and local object-storage volume.
2. Set `DATABASE_URL`, `JWT_SECRET`, and the existing storage configuration.
3. Deploy the `v0.2.1` image. Missing PostgreSQL migrations are applied automatically during startup.
4. Verify `/readyz` reports both database and storage as ready.

Desktop synchronization remains a separate release from the `synchub-desktop` repository.
