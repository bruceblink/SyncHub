# SyncHub v0.2.0 Release Notes

Date: 2026-07-12

SyncHub v0.2.0 moves the project to its current server and desktop architecture. The server is now PostgreSQL-only, includes the React administration console, and delegates end-user synchronization to SyncHub Desktop instead of shipping a CLI client.

## Highlights

- Added the React administration console for files, uploads, versions, activity, trash, devices, conflicts, and storage usage.
- Made PostgreSQL the only supported server metadata database and added embedded startup migrations.
- Added server readiness details for database and storage components.
- Added cloud file search, previews, chunked uploads, upload cancellation, version restore and pinning, and retained trash management.
- Added device synchronization health and conflict management APIs.
- Added storage quotas, expired upload cleanup, version retention, and safe unreferenced-object reclamation.
- Fixed upload idempotency so completed or expired sessions no longer block a later upload of the same content.
- Removed the retired CLI and client SDK from server release artifacts. SyncHub Desktop is now the supported synchronization client.

## Breaking Changes

- `DATABASE_URL` is required. SQLite server deployments are no longer supported.
- Server archives and Docker images no longer contain `synchub-cli`.
- Desktop synchronization is released separately from the `synchub-desktop` repository.

## Upgrade Notes

1. Back up the PostgreSQL database and local object-storage volume.
2. Set `DATABASE_URL`, `JWT_SECRET`, and the existing storage configuration.
3. Deploy the `v0.2.0` image. Missing PostgreSQL migrations are applied automatically during startup.
4. Verify `/readyz` reports both database and storage as ready.
5. Install SyncHub Desktop separately for end-user synchronization.

The upload idempotency migration replaces the old all-status unique index with a pending-session-only index. Existing file metadata and stored objects are preserved.
