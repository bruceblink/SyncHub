# SyncHub v0.1.0 MVP Release Notes

## Summary

SyncHub v0.1.0 is the first MVP release for a personal developer workspace sync loop. It focuses on a local-first setup: SQLite metadata, local filesystem object storage, REST API, CLI workflows, and an agent that can run one-shot or scheduled sync cycles.

## Highlights

- API server with health, readiness, version, metrics, Swagger UI, and OpenAPI YAML endpoints.
- SQLite-backed auth, file metadata, upload sessions, change feed, devices, conflicts, and version history.
- Local filesystem storage with chunk upload, compose, full download, range download, ETag, and conditional download support.
- CLI support for auth, workspace init, manifest scan, file list/download/mkdir/move/delete, upload status, sync push/pull/once/status/doctor/watch/trash/devices/conflicts, and server diagnostics.
- Agent support for scheduled sync, one-shot sync, dry-run mode, watch-triggered cycles, pause/resume, status, reset-state, and JSON output.
- Multi-device sync smoke coverage for register, workspace init, push, pull, agent control, version history, pin/unpin, restore, and download.
- Local backup and restore scripts for SQLite data plus object storage.
- Release scripts for multi-platform archives and SHA256 verification.

## Known MVP Scope

The v0.1.0 release intentionally keeps the product narrow:

- SQLite is the supported MVP database path.
- Local filesystem storage is the supported MVP object storage path.
- Docker Compose smoke testing is optional because registry access can be environment-dependent.
- WebDAV, S3/OSS/MinIO storage, team spaces, third-party adapters, and production PostgreSQL/MySQL adapters are deferred.
