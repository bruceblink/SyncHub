# SyncHub v0.1.1 Release Notes

Date: 2026-07-05

SyncHub v0.1.1 is an MVP patch release focused on the Docker-first Linux deployment path and the CLI daemon startup model.

## Highlights

- Folded the optional standalone agent into `synchub-cli sync daemon`, so clients no longer need a separate `synchub-agent` binary.
- `workspace init` now registers initialized workspaces in a user-level workspace registry.
- `synchub-cli sync daemon` can run without `--path` from a login/startup task and watches all registered workspaces for the current login config.
- `--path <workspace>` remains available for explicit single-workspace daemon operations.
- Added Fly.io deployment config and refreshed Docker image delivery docs for Linux server deployment.
- The runtime Docker image contains only the API server; `synchub-cli` ships through the auxiliary release archives.

## Upgrade Notes

Run the daemon from user login/startup with:

```bash
synchub-cli sync daemon
```

Existing workspaces should be registered once by rerunning:

```bash
synchub-cli workspace init --path <workspace> --remote-path <remote-path>
```

After that, the no-`--path` daemon command can discover the workspace automatically.
